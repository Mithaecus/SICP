1.1.4 Exercise 1.4

Observe that our model of evaluation allows for combinations whose operators are compound expressions. Use this observation to describe the behavior of the following procedure:

(define (a-plus-abs-b a b)
  ((if (> b 0) + -) a b))

If b is greater than 0, add a and b. Otherwise, substract b from a.

1.1.5 Exercise 1.5

How does the following code change if it’s ran using applicative-order evaluation vs normal-order evaluation?

(define (p) (p))
(define (test x y)
    (if (= x 0) 0 y))
 
(test 0 (p))

Normal-order evaluation:

(test 0 (p))
 
(if (= 0 0) 0 (p))
(p)
(p)
(p)
...

Normal-order fully expands and then reduces so when it reaches (p) it’ll expand an infinitely recusrive function meaning the interpreter will not finish.

Applicative-order evaluation:

(test 0 (p))
 
(if (= 0 0) 0 (p))
0

Applicative-order only evaluates what is required to run, so it’ll see (= 0 0) as #t and return without evaluating the infinite loop.
